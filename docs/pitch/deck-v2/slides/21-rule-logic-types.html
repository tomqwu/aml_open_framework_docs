Act II · The Spec Is The Contract

Four rule shapes, one discriminator.

Every rule's logic.type picks one of four shapes. Pydantic v2's discriminated union enforces it; the engine compiles each shape into the same alert contract.

Type · 01
aggregation_window
Count or sum thresholds over a sliding time window — the workhorse type.
# structuring type: aggregation_window window: 30d group_by: [customer_id] having: count: { gte: 3 } sum_amount: { gte: 20000 }
Use when: structuring · velocity · burst patterns · "N transactions totalling X within Y days."
Type · 02
custom_sql
Hand-written SQL for joins, sub-queries, or graph traversals the declarative form can't express.
# pass-through layering type: custom_sql sql: | SELECT customer_id FROM txns t1 JOIN txns t2 USING (counterparty_id) WHERE t1.window = '{window_start}' AND ABS(t1.amount - t2.amount) < 100 GROUP BY 1 HAVING COUNT(*) >= 3
Use when: complex joins · network traversal · placeholder substitution ({as_of}, {window_start}).
Type · 03
python_ref
Pluggable Python callable — escape hatch for ML scorers, anomaly models, custom logic.
# anomaly score type: python_ref module: scorers.iforest callable: score_anomaly model: id: iforest_v3.2 version: 2026.04 threshold: 0.85
Use when: ML scoring · custom features · the spec captures model_id + version for MRM under SR 26-2.
Type · 04
list_match
Customer or transaction is on a list — sanctions, PEP, internal watch, BOI control list.
# sanctions screening type: list_match list: ofac_sdn field: counterparty.name match: fuzzy threshold: 0.92 refresh: daily
Use when: sanctions / PEP / OFAC / EU consolidated · internal watchlists · BOI ownership-percent breaches.
One logic.type field. One Pydantic discriminator. One escape hatch when the abstraction isn't enough.
