Authentication
JWT on every request. OIDC for the issuer trust chain — paste in your IdP's discovery_url and the API picks up keys, issuer, audience automatically.
Multi-tenant isolation
The JWT's sub claim resolves to a tenant_id. Every query carries it — DuckDB views are scoped, audit ledger entries are scoped, examiner packs are scoped. Tenant A cannot see tenant B, even at the audit-trail layer.
Same answers, two surfaces
Dashboard and API share one engine. POST /runs produces the same alerts the dashboard shows, with the same output_hash. No drift between programmatic access and human review.
Same Manifest. Same engine. API or CLI — same answer, same hash.
