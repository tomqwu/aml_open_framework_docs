Act IV · The Detection Substrate

Same in, same out — published as a contract.

Every commercial platform produces operational logs. None publish a same-Manifest + same-data + same-seed = identical-output guarantee. Ours is enforced by a test that runs in CI on every commit.

spec_hash
SHA-256 of the Compliance Manifest — git SHA + content hash, captured at run start.
input_hash
Hash of the ordered input rows. The contract names which data snapshot was used.
engine_version
Pinned framework version — same engine commit replays produce the same answers.
seed
Deterministic seed — synthetic data and any randomised order is reproducible.
spec_hash + input_hash + engine_version + seed → output_hash
An auditor re-runs any historical execution and verifies the output hash matches. If it doesn't, the chain of custody is broken and the run is flagged.
tests/test_run_reproducibility.py  ·  runs in CI on every commit
# tests/test_run_reproducibility.py def test_run_is_reproducible(tmp_path): # Same Manifest, same data, same seed. spec = load_spec("examples/canadian_schedule_i_bank/aml.yaml") data = synthetic_dataset(seed=42) out_a = Engine(spec).run(data, seed=42) out_b = Engine(spec).run(data, seed=42) assert out_a.output_hash == out_b.output_hash assert out_a.decision_log_hash == out_b.decision_log_hash assert out_a.alerts == out_b.alerts # Mutation detection · change one byte in the spec spec_v2 = load_spec("...mutated.yaml") out_c = Engine(spec_v2).run(data, seed=42) assert out_c.spec_hash != out_a.spec_hash assert out_c.output_hash != out_a.output_hash # tampering caught
