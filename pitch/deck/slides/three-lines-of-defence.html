AML OPEN FRAMEWORK
09 / 27 · THREE LINES OF DEFENCE
Same org chart. Sharper handoffs.

One artifact, three roles, no overlap.

The framework doesn't redraw lines of defence — it gives each line the same source-of-truth (the spec, the run, the audit ledger) and lets each operate at the level OSFI's E-23 actually expects: 1LoD owns, 2LoD challenges, 3LoD assures.
1st
First Line · Owns the Risk

Build, run, defend the rules.

FCC Operations · Detection Engineers · L1/L2 Analysts
  • OwnsThe spec YAML — every detection rule, threshold, regulation citation
  • OperatesThe engine run — daily/intraday on production data
  • InvestigatesAlerts; files STRs with auto-attached evidence
  • MaintainsTuning Lab calibration evidence per rule
FRAMEWORK SURFACES
spec/rules/*.yaml · Alert Queue · Tuning Lab · Today landing
2nd
Second Line · Challenges

Read it, question it, sign it off.

Compliance · MLRO · Model Risk Mgmt · 2LoD QA
  • ReviewsSpec PRs — same artifact 1LoD ships; plain English
  • ValidatesPer-rule MRM dossier (SR 26-2 / Interagency)
  • ApprovesSTRs, threshold changes, model promotions
  • ReportsEffectiveness pack to board, regulator, auditor
FRAMEWORK SURFACES
git PR review · Effectiveness page · MRM bundle · STR review queue
3rd
Third Line · Assures

Replay it, verify it, attest.

Internal Audit · External Audit · OSFI / FINTRAC examiners
  • PullsHash-chained audit ledger for any historical period
  • Verifiesverify_decisions() — tampering = control failure
  • ReplaysAny run at its spec hash; bytes match
  • ExportsExamination ZIP — section-mapped to FINTRAC / OSFI
FRAMEWORK SURFACES
decisions.jsonl · Audit & Evidence page · aml audit-pack CLI
1LoD → 2LoD Handoff is a git PR, not a meeting deck. 2LoD reads the same YAML 1LoD ships. Diff-able. Reviewable. Signed.
2LoD → 3LoD Handoff is a hash + ledger pointer, not a binder. 3LoD replays the run; bytes match the attestation. Tamper-evident.
No more "compliance said one thing, IT did another." There is one artifact all three lines reference.
