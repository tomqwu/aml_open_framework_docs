What this actually is, no jargon

One file becomes your rule book,
your dashboard, and your regulator's audit trail.

Banks lose years to drift between policy (what the regulator wrote), implementation (what the pipelines actually do), and evidence (what the auditor can prove). This framework collapses all three into one editable YAML file. Edit the file → SQL queries, detection rules, the analyst dashboard, the audit ledger, and the SAR/STR export all regenerate from it — and every line traces back to a specific regulation clause.
EDIT ONCE
Change a threshold in the spec — every downstream artifact follows.
No "did engineering update the rules?" Slack threads.
REPLAY ANYTIME
Auditors verify by re-running the same spec at the same hash.
Mismatched output is a control failure — not a debate.
EXPORT IN ONE COMMAND
FINTRAC ZIP, AMLA RTS JSON, goAML XML — all on demand.
From regulator's announcement to compiled answer in < 60s.
THE ANALOGY
Think Terraform for compliance. You declare what you want once; the framework generates the queries, the dashboards, the audit trail — all signed, reproducible, reviewable.
