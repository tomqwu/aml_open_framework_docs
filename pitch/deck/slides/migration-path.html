AML OPEN FRAMEWORK
10 / 27 · MIGRATION PATH
No CTO will rip out their AML stack on a Tuesday.

You don't replace the engine. You wrap it, then graduate.

Four stages, ~12 months. Each stage is independently valuable — stop at any gate and you've still gained audit-grade evidence and a 2LoD-readable spec. The incumbent vendor stays online until you decide it doesn't need to.
Stage 1 Wk 0–8

Shadow — read-only mirror.

No production change. Auditor-grade evidence on day 30.
  • Point framework at same warehouse the incumbent reads
  • Translate top-20 incumbent rules into spec YAML
  • Run engine nightly; emit audit ledger
  • Compare alert sets vs incumbent — diff report weekly
Exit Gate
≥ 95% alert overlap on 90-day window. 2LoD signs off on top-20 spec.
Stage 2 Wk 8–20

Dual — both engines fire alerts.

Alerts merged into one queue. Disagreements become learning.
  • Both engines write to shared alert queue
  • Analyst sees provenance flag on each alert
  • Discrepancies drive spec tuning (Tuning Lab)
  • Framework runs net-new typologies incumbent can't
Exit Gate
Override rate parity. MRM bundle approved by 2LoD for in-scope rules.
Stage 3 Wk 20–40

Primary — framework leads.

Incumbent demoted to fallback. First exam runs on framework evidence.
  • Framework is system-of-record for in-scope typologies
  • Incumbent retained as fallback + edge cases
  • FINTRAC / OSFI examination ZIP generated, not assembled
  • Vendor licence renegotiated downward
Exit Gate
One full exam cycle with no incumbent dependency. Effectiveness pack delivered.
Stage 4 Wk 40+

Sole — vendor optional.

Incumbent kept only where business case demands. Most banks: gone.
  • All in-scope rules on framework
  • Vendor retained only for specialty modules (sanctions UI, etc.)
  • Net-new jurisdictions onboard in weeks, not quarters
  • Reinvest savings in detection R&D, not licence fees
Exit Gate
Annual TCO target met. $18–28M / yr mid-point reduction realized.
